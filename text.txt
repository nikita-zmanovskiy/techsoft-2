Общие требования:
Разработать Single Page Application (SPA) на React с использованием React Router. Приложение имитирует интерфёйс платёжной системы для создания новой выплаты.
Обязательные технологии:
•	React (функциональные компоненты, хуки)
•	React Router
•	Zustand (для управления состоянием)
•	Feature-Sliced Design (FSD) (архитектура проекта)
•	TypeScript (обязательно)

Описание задачи
Страница создания выплаты
1.	Зависимые поля формы:
o	Страна получателя (select):
	Варианты: США, Великобритания, Германия
	При выборе загружаются доступные валюты для выбранной страны
o	Валюта выплаты (select):
	Заблокировано, пока не выбрана страна
	Данные зависят от страны:
	США → Доллар (USD), Евро (EUR)
	Великобритания → Фунт стерлингов (GBP)
	Германия → Евро (EUR)
	При смене страны - поле сбрасывается
o	Способ выплаты (select):
	Заблокировано, пока не выбрана валюта
	Данные зависят от валюты:
	Доллар (USD) → Bank Account, PayPal
	Евро (EUR) → SEPA Transfer, Payoneer
	Фунт стерлингов (GBP) → Wise, UK Bank Transfer
	При смене валюты - поле сбрасывается
2.	Валидация:
o	Все поля обязательны для заполнения
o	Кнопка "Создать выплату" активна только при валидной форме
3.	Отправка формы:
o	При нажатии кнопки "Создать выплату":
	Показывать лоадер (опционально)
	Переходить на страницу /payment-results


Страница результатов
1.	Отображение данных:
o	Показать ранее введённые данные
2.	Навигация:
o	Кнопка "Создать новую выплату" для возврата на первую страницу

Дополнительные требования
1.	Мокапирование API:
o	Создать мокапы для загрузки валют и методов выплаты
o	Использовать setTimeout для симуляции задержки сети
2.	Стилизация (опционально):
o	CSS Modules или Styled Components для стилизации
